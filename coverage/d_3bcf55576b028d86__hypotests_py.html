25def epps_singleton_2samp(x, y, t=(0.4, 0.8)):

27 

28 Test the null hypothesis that two samples have the same underlying 

29 probability distribution. 

30 

31 Parameters 

32 ---------- 

33 x, y : array-like 

34 The two samples of observations to be tested. Input must not have more 

35 than one dimension. Samples can have different lengths. 

36 t : array-like, optional 

37 The points (t1, ..., tn) where the empirical characteristic function is 

38 to be evaluated. It should be positive distinct numbers. The default 

39 value (0.4, 0.8) is proposed in [1]_. Input must not have more than 

40 one dimension. 

41 

42 Returns 

43 ------- 

44 statistic : float 

45 The test statistic. 

46 pvalue : float 

47 The associated p-value based on the asymptotic chi2-distribution. 

48 

49 See Also 

50 -------- 

51 ks_2samp, anderson_ksamp 

52 

53 Notes 

54 ----- 

55 Testing whether two samples are generated by the same underlying 

56 distribution is a classical question in statistics. A widely used test is 

57 the Kolmogorov-Smirnov (KS) test which relies on the empirical 

58 distribution function. Epps and Singleton introduce a test based on the 

59 empirical characteristic function in [1]_. 

60 

61 One advantage of the ES test compared to the KS test is that is does 

62 not assume a continuous distribution. In [1]_, the authors conclude 

63 that the test also has a higher power than the KS test in many 

64 examples. They recommend the use of the ES test for discrete samples as 

65 well as continuous samples with at least 25 observations each, whereas 

66 `anderson_ksamp` is recommended for smaller sample sizes in the 

67 continuous case. 

68 

69 The p-value is computed from the asymptotic distribution of the test 

70 statistic which follows a `chi2` distribution. If the sample size of both 

71 `x` and `y` is below 25, the small sample correction proposed in [1]_ is 

72 applied to the test statistic. 

73 

74 The default values of `t` are determined in [1]_ by considering 

75 various distributions and finding good values that lead to a high power 

76 of the test in general. Table III in [1]_ gives the optimal values for 

77 the distributions tested in that study. The values of `t` are scaled by 

78 the semi-interquartile range in the implementation, see [1]_. 

79 

80 References 

81 ---------- 

82 .. [1] T. W. Epps and K. J. Singleton, "An omnibus test for the two-sample 

83 problem using the empirical characteristic function", Journal of 

84 Statistical Computation and Simulation 26, p. 177--203, 1986. 

85 

86 .. [2] S. J. Goerg and J. Kaiser, "Nonparametric testing of distributions 

87 - the Epps-Singleton two-sample test using the empirical characteristic 

88 function", The Stata Journal 9(3), p. 454--465, 2009. 

89

156def _psi1_mod(x): 

157 """ 

158 psi1 is defined in equation 1.10 in Csorgo, S. and Faraway, J. (1996). 

159 This implements a modified version by excluding the term V(x) / 12 

160 (here: _cdf_cvm_inf(x) / 12) to avoid evaluating _cdf_cvm_inf(x) 

161 twice in _cdf_cvm. 

162 

163 Implementation based on MAPLE code of Julian Faraway and R code of the 

164 function pCvM in the package goftest (v1.1.1), permission granted 

165 by Adrian Baddeley. Main difference in the implementation: the code 

166 here keeps adding terms of the series until the terms are small enough. 

167 """ 

168 

169 def _ed2(y): 

170 z = y**2 / 4 

171 b = kv(1/4, z) + kv(3/4, z) 

172 return np.exp(-z) * (y/2)**(3/2) * b / np.sqrt(np.pi) 

173 

174 def _ed3(y): 

175 z = y**2 / 4 

176 c = np.exp(-z) / np.sqrt(np.pi) 

177 return c * (y/2)**(5/2) * (2*kv(1/4, z) + 3*kv(3/4, z) - kv(5/4, z)) 

178 

179 def _Ak(k, x): 

180 m = 2*k + 1 

181 sx = 2 * np.sqrt(x) 

182 y1 = x**(3/4) 

183 y2 = x**(5/4) 

184 

185 e1 = m * gamma(k + 1/2) * _ed2((4 * k + 3)/sx) / (9 * y1) 

186 e2 = gamma(k + 1/2) * _ed3((4 * k + 1) / sx) / (72 * y2) 

187 e3 = 2 * (m + 2) * gamma(k + 3/2) * _ed3((4 * k + 5) / sx) / (12 * y2) 

188 e4 = 7 * m * gamma(k + 1/2) * _ed2((4 * k + 1) / sx) / (144 * y1) 

189 e5 = 7 * m * gamma(k + 1/2) * _ed2((4 * k + 5) / sx) / (144 * y1) 

190 

191 return e1 + e2 + e3 + e4 + e5 

192 

193 x = np.asarray(x) 

194 tot = np.zeros_like(x, dtype='float') 

195 cond = np.ones_like(x, dtype='bool') 

196 k = 0 

197 while np.any(cond): 

198 z = -_Ak(k, x[cond]) / (np.pi * gamma(k + 1)) 

199 tot[cond] = tot[cond] + z 

200 cond[cond] = np.abs(z) >= 1e-7 

201 k += 1 

202 

203 return tot

206def _cdf_cvm_inf(x): 

207 """ 

208 Calculate the cdf of the Cramér-von Mises statistic (infinite sample size). 

209 

210 See equation 1.2 in Csorgo, S. and Faraway, J. (1996). 

211 

212 Implementation based on MAPLE code of Julian Faraway and R code of the 

213 function pCvM in the package goftest (v1.1.1), permission granted 

214 by Adrian Baddeley. Main difference in the implementation: the code 

215 here keeps adding terms of the series until the terms are small enough. 

216 

217 The function is not expected to be accurate for large values of x, say 

218 x > 4, when the cdf is very close to 1. 

219 """ 

220 x = np.asarray(x) 

221 

222 def term(x, k): 

223 # this expression can be found in [2], second line of (1.3) 

224 u = np.exp(gammaln(k + 0.5) - gammaln(k+1)) / (np.pi**1.5 * np.sqrt(x)) 

225 y = 4*k + 1 

226 q = y**2 / (16*x) 

227 b = kv(0.25, q) 

228 return u * np.sqrt(y) * np.exp(-q) * b 

229 

230 tot = np.zeros_like(x, dtype='float') 

231 cond = np.ones_like(x, dtype='bool') 

232 k = 0 

233 while np.any(cond): 

234 z = term(x[cond], k) 

235 tot[cond] = tot[cond] + z 

236 cond[cond] = np.abs(z) >= 1e-7 

237 k += 1 

238 

239 return tot

242def _cdf_cvm(x, n=None): 

243 """ 

244 Calculate the cdf of the Cramér-von Mises statistic for a finite sample 

245 size n. If N is None, use the asymptotic cdf (n=inf). 

246 

247 See equation 1.8 in Csorgo, S. and Faraway, J. (1996) for finite samples, 

248 1.2 for the asymptotic cdf. 

249 

250 The function is not expected to be accurate for large values of x, say 

251 x > 2, when the cdf is very close to 1 and it might return values > 1 

252 in that case, e.g. _cdf_cvm(2.0, 12) = 1.0000027556716846. 

253 """ 

254 x = np.asarray(x) 

255 if n is None: 

256 y = _cdf_cvm_inf(x) 

257 else: 

258 # support of the test statistic is [12/n, n/3], see 1.1 in [2] 

259 y = np.zeros_like(x, dtype='float') 

260 sup = (1./(12*n) < x) & (x < n/3.) 

261 # note: _psi1_mod does not include the term _cdf_cvm_inf(x) / 12 

262 # therefore, we need to add it here 

263 y[sup] = _cdf_cvm_inf(x[sup]) * (1 + 1./(12*n)) + _psi1_mod(x[sup]) / n 

264 y[x >= n/3] = 1 

265 

266 if y.ndim == 0: 

267 return y[()] 

268 return y

271def cramervonmises(rvs, cdf, args=()): 

272 """Perform the one-sample Cramér-von Mises test for goodness of fit. 

273 

274 This performs a test of the goodness of fit of a cumulative distribution 

275 function (cdf) :math:`F` compared to the empirical distribution function 

276 :math:`F_n` of observed random variates :math:`X_1, ..., X_n` that are 

277 assumed to be independent and identically distributed ([1]_). 

278 The null hypothesis is that the :math:`X_i` have cumulative distribution 

279 :math:`F`. 

280 

281 Parameters 

282 ---------- 

283 rvs : array_like 

284 A 1-D array of observed values of the random variables :math:`X_i`. 

285 cdf : str or callable 

286 The cumulative distribution function :math:`F` to test the 

287 observations against. If a string, it should be the name of a 

288 distribution in `scipy.stats`. If a callable, that callable is used 

289 to calculate the cdf: ``cdf(x, *args) -> float``. 

290 args : tuple, optional 

291 Distribution parameters. These are assumed to be known; see Notes. 

292 

293 Returns 

294 ------- 

295 res : object with attributes 

296 statistic : float 

297 Cramér-von Mises statistic. 

298 pvalue : float 

299 The p-value. 

300 

301 See Also 

302 -------- 

303 kstest, cramervonmises_2samp 

304 

305 Notes 

306 ----- 

307 .. versionadded:: 1.6.0 

308 

309 The p-value relies on the approximation given by equation 1.8 in [2]_. 

310 It is important to keep in mind that the p-value is only accurate if 

311 one tests a simple hypothesis, i.e. the parameters of the reference 

312 distribution are known. If the parameters are estimated from the data 

313 (composite hypothesis), the computed p-value is not reliable. 

314 

315 References 

316 ---------- 

317 .. [1] Cramér-von Mises criterion, Wikipedia, 

318 https://en.wikipedia.org/wiki/Cram%C3%A9r%E2%80%93von_Mises_criterion 

319 .. [2] Csorgo, S. and Faraway, J. (1996). The Exact and Asymptotic 

320 Distribution of Cramér-von Mises Statistics. Journal of the 

321 Royal Statistical Society, pp. 221-234. 

322 

323 Examples 

324 -------- 

325 

326 Suppose we wish to test whether data generated by ``scipy.stats.norm.rvs`` 

327 were, in fact, drawn from the standard normal distribution. We choose a 

328 significance level of alpha=0.05. 

329 

330 >>> from scipy import stats 

331 >>> rng = np.random.default_rng() 

332 >>> x = stats.norm.rvs(size=500, random_state=rng) 

333 >>> res = stats.cramervonmises(x, 'norm') 

334 >>> res.statistic, res.pvalue 

335 (0.49121480855028343, 0.04189256516661377) 

336 

337 The p-value 0.79 exceeds our chosen significance level, so we do not 

338 reject the null hypothesis that the observed sample is drawn from the 

339 standard normal distribution. 

340 

341 Now suppose we wish to check whether the same samples shifted by 2.1 is 

342 consistent with being drawn from a normal distribution with a mean of 2. 

343 

344 >>> y = x + 2.1 

345 >>> res = stats.cramervonmises(y, 'norm', args=(2,)) 

346 >>> res.statistic, res.pvalue 

347 (0.07400330012187435, 0.7274595666160468) 

348 

349 Here we have used the `args` keyword to specify the mean (``loc``) 

350 of the normal distribution to test the data against. This is equivalent 

351 to the following, in which we create a frozen normal distribution with 

352 mean 2.1, then pass its ``cdf`` method as an argument. 

353 

354 >>> frozen_dist = stats.norm(loc=2) 

355 >>> res = stats.cramervonmises(y, frozen_dist.cdf) 

356 >>> res.statistic, res.pvalue 

357 (0.07400330012187435, 0.7274595666160468) 

358 

359 In either case, we would reject the null hypothesis that the observed 

360 sample is drawn from a normal distribution with a mean of 2 (and default 

361 variance of 1) because the p-value 0.04 is less than our chosen 

362 significance level. 

363 

364 """ 

365 if isinstance(cdf, str): 

366 cdf = getattr(distributions, cdf).cdf 

367 

368 vals = np.sort(np.asarray(rvs)) 

369 

370 if vals.size <= 1: 

371 raise ValueError('The sample must contain at least two observations.') 

372 if vals.ndim > 1: 

373 raise ValueError('The sample must be one-dimensional.') 

374 

375 n = len(vals) 

376 cdfvals = cdf(vals, *args) 

377 

378 u = (2*np.arange(1, n+1) - 1)/(2*n) 

379 w = 1/(12*n) + np.sum((u - cdfvals)**2) 

380 

381 # avoid small negative values that can occur due to the approximation 

382 p = max(0, 1. - _cdf_cvm(w, n)) 

383 

384 return CramerVonMisesResult(statistic=w, pvalue=p)

403def _tau_b(A): 

404 """Calculate Kendall's tau-b and p-value from contingency table.""" 

405 # See [2] 2.2 and 4.2 

406 

407 # contingency table must be truly 2D 

408 if A.shape[0] == 1 or A.shape[1] == 1: 

409 return np.nan, np.nan 

410 

411 NA = A.sum() 

412 PA = _P(A) 

413 QA = _Q(A) 

414 Sri2 = (A.sum(axis=1)**2).sum() 

415 Scj2 = (A.sum(axis=0)**2).sum() 

416 denominator = (NA**2 - Sri2)*(NA**2 - Scj2) 

417 

418 tau = (PA-QA)/(denominator)**0.5 

419 

420 numerator = 4*(_a_ij_Aij_Dij2(A) - (PA - QA)**2 / NA) 

421 s02_tau_b = numerator/denominator 

422 if s02_tau_b == 0: # Avoid divide by zero 

423 return tau, 0 

424 Z = tau/s02_tau_b**0.5 

425 p = 2*norm.sf(abs(Z)) # 2-sided p-value 

426 

427 return tau, p

430def _somers_d(A, alternative='two-sided'): 

431 """Calculate Somers' D and p-value from contingency table.""" 

432 # See [3] page 1740 

433 

434 # contingency table must be truly 2D 

435 if A.shape[0] <= 1 or A.shape[1] <= 1: 

436 return np.nan, np.nan 

437

439 NA2 = NA**2 

440 PA = _P(A)

460def somersd(x, y=None, alternative='two-sided'): 

461 r"""Calculates Somers' D, an asymmetric measure of ordinal association. 

462 

463 Like Kendall's :math:`\tau`, Somers' :math:`D` is a measure of the 

464 correspondence between two rankings. Both statistics consider the 

465 difference between the number of concordant and discordant pairs in two 

466 rankings :math:`X` and :math:`Y`, and both are normalized such that values 

467 close to 1 indicate strong agreement and values close to -1 indicate 

468 strong disagreement. They differ in how they are normalized. To show the 

469 relationship, Somers' :math:`D` can be defined in terms of Kendall's 

470 :math:`\tau_a`: 

471 

472 .. math:: 

473 D(Y|X) = \frac{\tau_a(X, Y)}{\tau_a(X, X)} 

474 

475 Suppose the first ranking :math:`X` has :math:`r` distinct ranks and the 

476 second ranking :math:`Y` has :math:`s` distinct ranks. These two lists of 

477 :math:`n` rankings can also be viewed as an :math:`r \times s` contingency 

478 table in which element :math:`i, j` is the number of rank pairs with rank 

479 :math:`i` in ranking :math:`X` and rank :math:`j` in ranking :math:`Y`. 

480 Accordingly, `somersd` also allows the input data to be supplied as a 

481 single, 2D contingency table instead of as two separate, 1D rankings. 

482 

483 Note that the definition of Somers' :math:`D` is asymmetric: in general, 

484 :math:`D(Y|X) \neq D(X|Y)`. ``somersd(x, y)`` calculates Somers' 

485 :math:`D(Y|X)`: the "row" variable :math:`X` is treated as an independent 

486 variable, and the "column" variable :math:`Y` is dependent. For Somers' 

487 :math:`D(X|Y)`, swap the input lists or transpose the input table. 

488 

489 Parameters 

490 ---------- 

491 x: array_like 

492 1D array of rankings, treated as the (row) independent variable. 

493 Alternatively, a 2D contingency table. 

494 y: array_like, optional 

495 If `x` is a 1D array of rankings, `y` is a 1D array of rankings of the 

496 same length, treated as the (column) dependent variable. 

497 If `x` is 2D, `y` is ignored. 

498 alternative : {'two-sided', 'less', 'greater'}, optional 

499 Defines the alternative hypothesis. Default is 'two-sided'. 

500 The following options are available: 

501 * 'two-sided': the rank correlation is nonzero 

502 * 'less': the rank correlation is negative (less than zero) 

503 * 'greater': the rank correlation is positive (greater than zero) 

504 

505 Returns 

506 ------- 

507 res : SomersDResult 

508 A `SomersDResult` object with the following fields: 

509 

510 correlation : float 

511 The Somers' :math:`D` statistic. 

512 pvalue : float 

513 The p-value for a hypothesis test whose null 

514 hypothesis is an absence of association, :math:`D=0`. 

515 See notes for more information. 

516 table : 2D array 

517 The contingency table formed from rankings `x` and `y` (or the 

518 provided contingency table, if `x` is a 2D array) 

519 

520 See Also 

521 -------- 

522 kendalltau : Calculates Kendall's tau, another correlation measure. 

523 weightedtau : Computes a weighted version of Kendall's tau. 

524 spearmanr : Calculates a Spearman rank-order correlation coefficient. 

525 pearsonr : Calculates a Pearson correlation coefficient. 

526 

527 Notes 

528 ----- 

529 This function follows the contingency table approach of [2]_ and 

530 [3]_. *p*-values are computed based on an asymptotic approximation of 

531 the test statistic distribution under the null hypothesis :math:`D=0`. 

532 

533 Theoretically, hypothesis tests based on Kendall's :math:`tau` and Somers' 

534 :math:`D` should be identical. 

535 However, the *p*-values returned by `kendalltau` are based 

536 on the null hypothesis of *independence* between :math:`X` and :math:`Y` 

537 (i.e. the population from which pairs in :math:`X` and :math:`Y` are 

538 sampled contains equal numbers of all possible pairs), which is more 

539 specific than the null hypothesis :math:`D=0` used here. If the null 

540 hypothesis of independence is desired, it is acceptable to use the 

541 *p*-value returned by `kendalltau` with the statistic returned by 

542 `somersd` and vice versa. For more information, see [2]_. 

543 

544 Contingency tables are formatted according to the convention used by 

545 SAS and R: the first ranking supplied (``x``) is the "row" variable, and 

546 the second ranking supplied (``y``) is the "column" variable. This is 

547 opposite the convention of Somers' original paper [1]_. 

548 

549 References 

550 ---------- 

551 .. [1] Robert H. Somers, "A New Asymmetric Measure of Association for 

552 Ordinal Variables", *American Sociological Review*, Vol. 27, No. 6, 

553 pp. 799--811, 1962.

664def barnard_exact(table, alternative="two-sided", pooled=True, n=32): 

665 r"""Perform a Barnard exact test on a 2x2 contingency table. 

666 

667 Parameters 

668 ---------- 

669 table : array_like of ints 

670 A 2x2 contingency table. Elements should be non-negative integers. 

671 

672 alternative : {'two-sided', 'less', 'greater'}, optional 

673 Defines the null and alternative hypotheses. Default is 'two-sided'. 

674 Please see explanations in the Notes section below. 

675 

676 pooled : bool, optional 

677 Whether to compute score statistic with pooled variance (as in 

678 Student's t-test, for example) or unpooled variance (as in Welch's 

679 t-test). Default is ``True``. 

680 

681 n : int, optional 

682 Number of sampling points used in the construction of the sampling 

683 method. Note that this argument will automatically be converted to 

684 the next higher power of 2 since `scipy.stats.qmc.Sobol` is used to 

685 select sample points. Default is 32. Must be positive. In most cases, 

686 32 points is enough to reach good precision. More points comes at 

687 performance cost. 

688 

689 Returns 

690 ------- 

691 ber : BarnardExactResult 

692 A result object with the following attributes. 

693 

694 statistic : float 

695 The Wald statistic with pooled or unpooled variance, depending 

696 on the user choice of `pooled`. 

697 

698 pvalue : float 

699 P-value, the probability of obtaining a distribution at least as 

700 extreme as the one that was actually observed, assuming that the 

701 null hypothesis is true. 

702 

703 See Also 

704 -------- 

705 chi2_contingency : Chi-square test of independence of variables in a 

706 contingency table. 

707 fisher_exact : Fisher exact test on a 2x2 contingency table. 

708 boschloo_exact : Boschloo's exact test on a 2x2 contingency table, 

709 which is an uniformly more powerful alternative to Fisher's exact test. 

710 

711 Notes 

712 ----- 

713 Barnard's test is an exact test used in the analysis of contingency 

714 tables. It examines the association of two categorical variables, and 

715 is a more powerful alternative than Fisher's exact test 

716 for 2x2 contingency tables. 

717 

718 Let's define :math:`X_0` a 2x2 matrix representing the observed sample, 

719 where each column stores the binomial experiment, as in the example 

720 below. Let's also define :math:`p_1, p_2` the theoretical binomial 

721 probabilities for :math:`x_{11}` and :math:`x_{12}`. When using 

722 Barnard exact test, we can assert three different null hypotheses : 

723 

724 - :math:`H_0 : p_1 \geq p_2` versus :math:`H_1 : p_1 < p_2`, 

725 with `alternative` = "less" 

726 

727 - :math:`H_0 : p_1 \leq p_2` versus :math:`H_1 : p_1 > p_2`, 

728 with `alternative` = "greater" 

729 

730 - :math:`H_0 : p_1 = p_2` versus :math:`H_1 : p_1 \neq p_2`, 

731 with `alternative` = "two-sided" (default one) 

732 

733 In order to compute Barnard's exact test, we are using the Wald 

734 statistic [3]_ with pooled or unpooled variance. 

735 Under the default assumption that both variances are equal 

736 (``pooled = True``), the statistic is computed as: 

737 

738 .. math:: 

739 

740 T(X) = \frac{ 

741 \hat{p}_1 - \hat{p}_2 

742 }{ 

743 \sqrt{ 

744 \hat{p}(1 - \hat{p}) 

745 (\frac{1}{c_1} + 

746 \frac{1}{c_2}) 

747 } 

748 } 

749 

750 with :math:`\hat{p}_1, \hat{p}_2` and :math:`\hat{p}` the estimator of 

751 :math:`p_1, p_2` and :math:`p`, the latter being the combined probability, 

752 given the assumption that :math:`p_1 = p_2`. 

753 

754 If this assumption is invalid (``pooled = False``), the statistic is: 

755 

756 .. math:: 

757 

758 T(X) = \frac{ 

759 \hat{p}_1 - \hat{p}_2 

760 }{ 

761 \sqrt{ 

762 \frac{\hat{p}_1 (1 - \hat{p}_1)}{c_1} + 

763 \frac{\hat{p}_2 (1 - \hat{p}_2)}{c_2} 

764 } 

765 } 

766 

767 The p-value is then computed as: 

768 

769 .. math:: 

770 

771 \sum 

772 \binom{c_1}{x_{11}} 

773 \binom{c_2}{x_{12}} 

774 \pi^{x_{11} + x_{12}} 

775 (1 - \pi)^{t - x_{11} - x_{12}} 

776 

777 where the sum is over all 2x2 contingency tables :math:`X` such that: 

778 * :math:`T(X) \leq T(X_0)` when `alternative` = "less", 

779 * :math:`T(X) \geq T(X_0)` when `alternative` = "greater", or 

780 * :math:`T(X) \geq |T(X_0)|` when `alternative` = "two-sided". 

781 Above, :math:`c_1, c_2` are the sum of the columns 1 and 2, 

782 and :math:`t` the total (sum of the 4 sample's element). 

783 

784 The returned p-value is the maximum p-value taken over the nuisance 

785 parameter :math:`\pi`, where :math:`0 \leq \pi \leq 1`. 

786 

787 This function's complexity is :math:`O(n c_1 c_2)`, where `n` is the 

788 number of sample points. 

789 

790 References 

791 ---------- 

792 .. [1] Barnard, G. A. "Significance Tests for 2x2 Tables". *Biometrika*. 

793 34.1/2 (1947): 123-138. :doi:`dpgkg3` 

794 

795 .. [2] Mehta, Cyrus R., and Pralay Senchaudhuri. "Conditional versus 

796 unconditional exact tests for comparing two binomials." 

797 *Cytel Software Corporation* 675 (2003): 1-5. 

798 

799 .. [3] "Wald Test". *Wikipedia*. https://en.wikipedia.org/wiki/Wald_test 

800 

801 Examples 

802 -------- 

803 An example use of Barnard's test is presented in [2]_. 

804 

805 Consider the following example of a vaccine efficacy study 

806 (Chan, 1998). In a randomized clinical trial of 30 subjects, 15 were 

807 inoculated with a recombinant DNA influenza vaccine and the 15 were 

808 inoculated with a placebo. Twelve of the 15 subjects in the placebo 

809 group (80%) eventually became infected with influenza whereas for the 

810 vaccine group, only 7 of the 15 subjects (47%) became infected. The 

811 data are tabulated as a 2 x 2 table:: 

812 

813 Vaccine Placebo 

814 Yes 7 12 

815 No 8 3 

816 

817 When working with statistical hypothesis testing, we usually use a 

818 threshold probability or significance level upon which we decide 

819 to reject the null hypothesis :math:`H_0`. Suppose we choose the common 

820 significance level of 5%. 

821 

822 Our alternative hypothesis is that the vaccine will lower the chance of 

823 becoming infected with the virus; that is, the probability :math:`p_1` of 

824 catching the virus with the vaccine will be *less than* the probability 

825 :math:`p_2` of catching the virus without the vaccine. Therefore, we call 

826 `barnard_exact` with the ``alternative="less"`` option: 

827 

828 >>> import scipy.stats as stats 

829 >>> res = stats.barnard_exact([[7, 12], [8, 3]], alternative="less") 

830 >>> res.statistic 

831 -1.894... 

832 >>> res.pvalue 

833 0.03407... 

834 

835 Under the null hypothesis that the vaccine will not lower the chance of 

836 becoming infected, the probability of obtaining test results at least as 

837 extreme as the observed data is approximately 3.4%. Since this p-value is 

838 less than our chosen significance level, we have evidence to reject 

839 :math:`H_0` in favor of the alternative. 

840 

841 Suppose we had used Fisher's exact test instead: 

842 

843 >>> _, pvalue = stats.fisher_exact([[7, 12], [8, 3]], alternative="less") 

844 >>> pvalue 

845 0.0640... 

846 

847 With the same threshold significance of 5%, we would not have been able 

848 to reject the null hypothesis in favor of the alternative. As stated in 

849 [2]_, Barnard's test is uniformly more powerful than Fisher's exact test 

850 because Barnard's test does not condition on any margin. Fisher's test 

851 should only be used when both sets of marginals are fixed. 

852 

853 """ 

854 if n <= 0: 

855 raise ValueError( 

856 "Number of points `n` must be strictly positive, " 

857 f"found {n!r}" 

858 ) 

859 

860 table = np.asarray(table, dtype=np.int64) 

861 

862 if not table.shape == (2, 2): 

863 raise ValueError("The input `table` must be of shape (2, 2).") 

864 

865 if np.any(table < 0): 

866 raise ValueError("All values in `table` must be nonnegative.") 

867 

868 if 0 in table.sum(axis=0): 

869 # If both values in column are zero, the p-value is 1 and 

870 # the score's statistic is NaN. 

871 return BarnardExactResult(np.nan, 1.0) 

872 

873 total_col_1, total_col_2 = table.sum(axis=0) 

874 

875 x1 = np.arange(total_col_1 + 1, dtype=np.int64).reshape(-1, 1) 

876 x2 = np.arange(total_col_2 + 1, dtype=np.int64).reshape(1, -1) 

877 

878 # We need to calculate the wald statistics for each combination of x1 and 

879 # x2. 

880 p1, p2 = x1 / total_col_1, x2 / total_col_2 

881 

882 if pooled: 

883 p = (x1 + x2) / (total_col_1 + total_col_2) 

884 variances = p * (1 - p) * (1 / total_col_1 + 1 / total_col_2) 

885 else: 

886 variances = p1 * (1 - p1) / total_col_1 + p2 * (1 - p2) / total_col_2 

887 

888 # To avoid warning when dividing by 0 

889 with np.errstate(divide="ignore", invalid="ignore"): 

890 wald_statistic = np.divide((p1 - p2), np.sqrt(variances)) 

891 

892 wald_statistic[p1 == p2] = 0 # Removing NaN values 

893 

894 wald_stat_obs = wald_statistic[table[0, 0], table[0, 1]] 

895 

896 if alternative == "two-sided": 

897 index_arr = np.abs(wald_statistic) >= abs(wald_stat_obs) 

898 elif alternative == "less": 

899 index_arr = wald_statistic <= wald_stat_obs 

900 elif alternative == "greater": 

901 index_arr = wald_statistic >= wald_stat_obs 

902 else: 

903 msg = ( 

904 "`alternative` should be one of {'two-sided', 'less', 'greater'}," 

905 f" found {alternative!r}" 

906 ) 

907 raise ValueError(msg) 

908 

909 x1_sum_x2 = x1 + x2 

910 

911 x1_log_comb = _compute_log_combinations(total_col_1) 

912 x2_log_comb = _compute_log_combinations(total_col_2) 

913 x1_sum_x2_log_comb = x1_log_comb[x1] + x2_log_comb[x2] 

914 

915 result = shgo( 

916 _get_binomial_log_p_value_with_nuisance_param, 

917 args=(x1_sum_x2, x1_sum_x2_log_comb, index_arr), 

918 bounds=((0, 1),), 

919 n=n, 

920 sampling_method="sobol", 

921 ) 

922 

923 # result.fun is the negative log pvalue and therefore needs to be 

924 # changed before return 

925 p_value = np.clip(np.exp(-result.fun), a_min=0, a_max=1) 

926 return BarnardExactResult(wald_stat_obs, p_value)

1255def cramervonmises_2samp(x, y, method='auto'): 

1256 """Perform the two-sample Cramér-von Mises test for goodness of fit. 

1257 

1258 This is the two-sample version of the Cramér-von Mises test ([1]_): 

1259 for two independent samples :math:`X_1, ..., X_n` and 

1260 :math:`Y_1, ..., Y_m`, the null hypothesis is that the samples 

1261 come from the same (unspecified) continuous distribution. 

1262 

1263 Parameters 

1264 ---------- 

1265 x : array_like 

1266 A 1-D array of observed values of the random variables :math:`X_i`. 

1267 y : array_like 

1268 A 1-D array of observed values of the random variables :math:`Y_i`. 

1269 method : {'auto', 'asymptotic', 'exact'}, optional 

1270 The method used to compute the p-value, see Notes for details. 

1271 The default is 'auto'. 

1272 

1273 Returns 

1274 ------- 

1275 res : object with attributes 

1276 statistic : float 

1277 Cramér-von Mises statistic. 

1278 pvalue : float 

1279 The p-value. 

1280 

1281 See Also 

1282 -------- 

1283 cramervonmises, anderson_ksamp, epps_singleton_2samp, ks_2samp 

1284 

1285 Notes 

1286 ----- 

1287 .. versionadded:: 1.7.0 

1288 

1289 The statistic is computed according to equation 9 in [2]_. The 

1290 calculation of the p-value depends on the keyword `method`: 

1291 

1292 - ``asymptotic``: The p-value is approximated by using the limiting 

1293 distribution of the test statistic. 

1294 - ``exact``: The exact p-value is computed by enumerating all 

1295 possible combinations of the test statistic, see [2]_. 

1296 

1297 The exact calculation will be very slow even for moderate sample 

1298 sizes as the number of combinations increases rapidly with the 

1299 size of the samples. If ``method=='auto'``, the exact approach 

1300 is used if both samples contain less than 10 observations, 

1301 otherwise the asymptotic distribution is used. 

1302 

1303 If the underlying distribution is not continuous, the p-value is likely to 

1304 be conservative (Section 6.2 in [3]_). When ranking the data to compute 

1305 the test statistic, midranks are used if there are ties. 

1306 

1307 References 

1308 ---------- 

1309 .. [1] https://en.wikipedia.org/wiki/Cramer-von_Mises_criterion 

1310 .. [2] Anderson, T.W. (1962). On the distribution of the two-sample 

1311 Cramer-von-Mises criterion. The Annals of Mathematical 

1312 Statistics, pp. 1148-1159. 

1313 .. [3] Conover, W.J., Practical Nonparametric Statistics, 1971. 

1314 

1315 Examples 

1316 -------- 

1317 

1318 Suppose we wish to test whether two samples generated by 

1319 ``scipy.stats.norm.rvs`` have the same distribution. We choose a 

1320 significance level of alpha=0.05. 

1321 

1322 >>> from scipy import stats 

1323 >>> rng = np.random.default_rng() 

1324 >>> x = stats.norm.rvs(size=100, random_state=rng) 

1325 >>> y = stats.norm.rvs(size=70, random_state=rng) 

1326 >>> res = stats.cramervonmises_2samp(x, y) 

1327 >>> res.statistic, res.pvalue 

1328 (0.29376470588235293, 0.1412873014573014) 

1329 

1330 The p-value exceeds our chosen significance level, so we do not 

1331 reject the null hypothesis that the observed samples are drawn from the 

1332 same distribution. 

1333 

1334 For small sample sizes, one can compute the exact p-values: 

1335 

1336 >>> x = stats.norm.rvs(size=7, random_state=rng) 

1337 >>> y = stats.t.rvs(df=2, size=6, random_state=rng) 

1338 >>> res = stats.cramervonmises_2samp(x, y, method='exact') 

1339 >>> res.statistic, res.pvalue 

1340 (0.197802197802198, 0.31643356643356646) 

1341 

1342 The p-value based on the asymptotic distribution is a good approximation 

1343 even though the sample size is small. 

1344 

1345 >>> res = stats.cramervonmises_2samp(x, y, method='asymptotic') 

1346 >>> res.statistic, res.pvalue 

1347 (0.197802197802198, 0.2966041181527128) 

1348 

1349 Independent of the method, one would not reject the null hypothesis at the 

1350 chosen significance level in this example. 

1351 

1352 """ 

1353 xa = np.sort(np.asarray(x)) 

1354 ya = np.sort(np.asarray(y)) 

1355 

1356 if xa.size <= 1 or ya.size <= 1: 

1357 raise ValueError('x and y must contain at least two observations.') 

1358 if xa.ndim > 1 or ya.ndim > 1: 

1359 raise ValueError('The samples must be one-dimensional.') 

1360 if method not in ['auto', 'exact', 'asymptotic']: 

1361 raise ValueError('method must be either auto, exact or asymptotic.') 

1362 

1363 nx = len(xa) 

1364 ny = len(ya) 

1365 

1366 if method == 'auto': 

1367 if max(nx, ny) > 10: 

1368 method = 'asymptotic' 

1369 else: 

1370 method = 'exact' 

1371 

1372 # get ranks of x and y in the pooled sample 

1373 z = np.concatenate([xa, ya]) 

1374 # in case of ties, use midrank (see [1]) 

1375 r = scipy.stats.rankdata(z, method='average') 

1376 rx = r[:nx] 

1377 ry = r[nx:] 

1378 

1379 # compute U (eq. 10 in [2]) 

1380 u = nx * np.sum((rx - np.arange(1, nx+1))**2) 

1381 u += ny * np.sum((ry - np.arange(1, ny+1))**2) 

1382 

1383 # compute T (eq. 9 in [2]) 

1384 k, N = nx*ny, nx + ny 

1385 t = u / (k*N) - (4*k - 1)/(6*N) 

1386 

1387 if method == 'exact': 

1388 p = _pval_cvm_2samp_exact(u, nx, ny) 

1389 else: 

1390 # compute expected value and variance of T (eq. 11 and 14 in [2]) 

1391 et = (1 + 1/N)/6 

1392 vt = (N+1) * (4*k*N - 3*(nx**2 + ny**2) - 2*k) 

1393 vt = vt / (45 * N**2 * 4 * k) 

1394 

1395 # computed the normalized statistic (eq. 15 in [2]) 

1396 tn = 1/6 + (t - et) / np.sqrt(45 * vt) 

1397 

1398 # approximate distribution of tn with limiting distribution 

1399 # of the one-sample test statistic 

1400 # if tn < 0.003, the _cdf_cvm_inf(tn) < 1.28*1e-18, return 1.0 directly 

1401 if tn < 0.003: 

1402 p = 1.0 

1403 else: 

1404 p = max(0, 1. - _cdf_cvm_inf(tn)) 

1405 

1406 return CramerVonMisesResult(statistic=t, pvalue=p)

1724def permutation_test(data, statistic, *, permutation_type='independent', 

1725 vectorized=False, n_resamples=9999, batch=None, 

1726 alternative="two-sided", axis=0, random_state=None): 

1727 r""" 

1728 Performs a permutation test of a given statistic on provided data. 

1729 

1730 For independent sample statistics, the null hypothesis is that the data are 

1731 randomly sampled from the same distribution. 

1732 For paired sample statistics, two null hypothesis can be tested: 

1733 that the data are paired at random or that the data are assigned to samples 

1734 at random. 

1735 

1736 Parameters 

1737 ---------- 

1738 data : iterable of array-like 

1739 Contains the samples, each of which is an array of observations. 

1740 Dimensions of sample arrays must be compatible for broadcasting except 

1741 along `axis`. 

1742 statistic : callable 

1743 Statistic for which the p-value of the hypothesis test is to be 

1744 calculated. `statistic` must be a callable that accepts samples 

1745 as separate arguments (e.g. ``statistic(*data)``) and returns the 

1746 resulting statistic. 

1747 If `vectorized` is set ``True``, `statistic` must also accept a keyword 

1748 argument `axis` and be vectorized to compute the statistic along the 

1749 provided `axis` of the sample arrays. 

1750 permutation_type : {'independent', 'samples', 'pairings'}, optional 

1751 The type of permutations to be performed, in accordance with the 

1752 null hypothesis. The first two permutation types are for paired sample 

1753 statistics, in which all samples contain the same number of 

1754 observations and observations with corresponding indices along `axis` 

1755 are considered to be paired; the third is for independent sample 

1756 statistics. 

1757 

1758 - ``'samples'`` : observations are assigned to different samples 

1759 but remain paired with the same observations from other samples. 

1760 This permutation type is appropriate for paired sample hypothesis 

1761 tests such as the Wilcoxon signed-rank test and the paired t-test. 

1762 - ``'pairings'`` : observations are paired with different observations, 

1763 but they remain within the same sample. This permutation type is 

1764 appropriate for association/correlation tests with statistics such 

1765 as Spearman's :math:`\rho`, Kendall's :math:`\tau`, and Pearson's 

1766 :math:`r`. 

1767 - ``'independent'`` (default) : observations are assigned to different 

1768 samples. Samples may contain different numbers of observations. This 

1769 permutation type is appropriate for independent sample hypothesis 

1770 tests such as the Mann-Whitney :math:`U` test and the independent 

1771 sample t-test. 

1772 

1773 Please see the Notes section below for more detailed descriptions 

1774 of the permutation types. 

1775 

1776 vectorized : bool, default: ``False`` 

1777 By default, `statistic` is assumed to calculate the statistic only for 

1778 1D arrays contained in `data`. If `vectorized` is set ``True``, 

1779 `statistic` must also accept a keyword argument `axis` and be 

1780 vectorized to compute the statistic along the provided `axis` of the ND 

1781 arrays in `data`. Use of a vectorized statistic can reduce computation 

1782 time. 

1783 n_resamples : int or np.inf, default: 9999 

1784 Number of random permutations (resamples) used to approximate the null 

1785 distribution. If greater than or equal to the number of distinct 

1786 permutations, the exact null distribution will be computed. 

1787 Note that the number of distinct permutations grows very rapidly with 

1788 the sizes of samples, so exact tests are feasible only for very small 

1789 data sets. 

1790 batch : int, optional 

1791 The number of permutations to process in each call to `statistic`. 

1792 Memory usage is O(`batch`*``n``), where ``n`` is the total size 

1793 of all samples, regardless of the value of `vectorized`. Default is 

1794 ``None``, in which case ``batch`` is the number of permutations. 

1795 alternative : {'two-sided', 'less', 'greater'}, optional 

1796 The alternative hypothesis for which the p-value is calculated. 

1797 For each alternative, the p-value is defined for exact tests as 

1798 follows. 

1799 

1800 - ``'greater'`` : the percentage of the null distribution that is 

1801 greater than or equal to the observed value of the test statistic. 

1802 - ``'less'`` : the percentage of the null distribution that is 

1803 less than or equal to the observed value of the test statistic. 

1804 - ``'two-sided'`` (default) : twice the smaller of the p-values above. 

1805 

1806 Note that p-values for randomized tests are calculated according to the 

1807 conservative (over-estimated) approximation suggested in [2]_ and [3]_ 

1808 rather than the unbiased estimator suggested in [4]_. That is, when 

1809 calculating the proportion of the randomized null distribution that is 

1810 as extreme as the observed value of the test statistic, the values in 

1811 the numerator and denominator are both increased by one. An 

1812 interpretation of this adjustment is that the observed value of the 

1813 test statistic is always included as an element of the randomized 

1814 null distribution. 

1815 The convention used for two-sided p-values is not universal; 

1816 the observed test statistic and null distribution are returned in 

1817 case a different definition is preferred. 

1818 

1819 axis : int, default: 0 

1820 The axis of the (broadcasted) samples over which to calculate the 

1821 statistic. If samples have a different number of dimensions, 

1822 singleton dimensions are prepended to samples with fewer dimensions 

1823 before `axis` is considered. 

1824 random_state : {None, int, `numpy.random.Generator`, 

1825 `numpy.random.RandomState`}, optional 

1826 

1827 Pseudorandom number generator state used to generate permutations. 

1828 

1829 If `random_state` is ``None`` (default), the 

1830 `numpy.random.RandomState` singleton is used. 

1831 If `random_state` is an int, a new ``RandomState`` instance is used, 

1832 seeded with `random_state`. 

1833 If `random_state` is already a ``Generator`` or ``RandomState`` 

1834 instance then that instance is used. 

1835 

1836 Returns 

1837 ------- 

1838 statistic : float or ndarray 

1839 The observed test statistic of the data. 

1840 pvalue : float or ndarray 

1841 The p-value for the given alternative. 

1842 null_distribution : ndarray 

1843 The values of the test statistic generated under the null hypothesis. 

1844 

1845 Notes 

1846 ----- 

1847 

1848 The three types of permutation tests supported by this function are 

1849 described below. 

1850 

1851 **Unpaired statistics** (``permutation_type='independent'``): 

1852 

1853 The null hypothesis associated with this permutation type is that all 

1854 observations are sampled from the same underlying distribution and that 

1855 they have been assigned to one of the samples at random. 

1856 

1857 Suppose ``data`` contains two samples; e.g. ``a, b = data``. 

1858 When ``1 < n_resamples < binom(n, k)``, where 

1859 

1860 * ``k`` is the number of observations in ``a``, 

1861 * ``n`` is the total number of observations in ``a`` and ``b``, and 

1862 * ``binom(n, k)`` is the binomial coefficient (``n`` choose ``k``), 

1863 

1864 the data are pooled (concatenated), randomly assigned to either the first 

1865 or second sample, and the statistic is calculated. This process is 

1866 performed repeatedly, `permutation` times, generating a distribution of the 

1867 statistic under the null hypothesis. The statistic of the original 

1868 data is compared to this distribution to determine the p-value. 

1869 

1870 When ``n_resamples >= binom(n, k)``, an exact test is performed: the data 

1871 are *partitioned* between the samples in each distinct way exactly once, 

1872 and the exact null distribution is formed. 

1873 Note that for a given partitioning of the data between the samples, 

1874 only one ordering/permutation of the data *within* each sample is 

1875 considered. For statistics that do not depend on the order of the data 

1876 within samples, this dramatically reduces computational cost without 

1877 affecting the shape of the null distribution (because the frequency/count 

1878 of each value is affected by the same factor). 

1879 

1880 For ``a = [a1, a2, a3, a4]`` and ``b = [b1, b2, b3]``, an example of this 

1881 permutation type is ``x = [b3, a1, a2, b2]`` and ``y = [a4, b1, a3]``. 

1882 Because only one ordering/permutation of the data *within* each sample 

1883 is considered in an exact test, a resampling like ``x = [b3, a1, b2, a2]`` 

1884 and ``y = [a4, a3, b1]`` would *not* be considered distinct from the 

1885 example above. 

1886 

1887 ``permutation_type='independent'`` does not support one-sample statistics, 

1888 but it can be applied to statistics with more than two samples. In this 

1889 case, if ``n`` is an array of the number of observations within each 

1890 sample, the number of distinct partitions is:: 

1891 

1892 np.product([binom(sum(n[i:]), sum(n[i+1:])) for i in range(len(n)-1)]) 

1893 

1894 **Paired statistics, permute pairings** (``permutation_type='pairings'``): 

1895 

1896 The null hypothesis associated with this permutation type is that 

1897 observations within each sample are drawn from the same underlying 

1898 distribution and that pairings with elements of other samples are 

1899 assigned at random. 

1900 

1901 Suppose ``data`` contains only one sample; e.g. ``a, = data``, and we 

1902 wish to consider all possible pairings of elements of ``a`` with elements 

1903 of a second sample, ``b``. Let ``n`` be the number of observations in 

1904 ``a``, which must also equal the number of observations in ``b``. 

1905 

1906 When ``1 < n_resamples < factorial(n)``, the elements of ``a`` are 

1907 randomly permuted. The user-supplied statistic accepts one data argument, 

1908 say ``a_perm``, and calculates the statistic considering ``a_perm`` and 

1909 ``b``. This process is performed repeatedly, `permutation` times, 

1910 generating a distribution of the statistic under the null hypothesis. 

1911 The statistic of the original data is compared to this distribution to 

1912 determine the p-value. 

1913 

1914 When ``n_resamples >= factorial(n)``, an exact test is performed: 

1915 ``a`` is permuted in each distinct way exactly once. Therefore, the 

1916 `statistic` is computed for each unique pairing of samples between ``a`` 

1917 and ``b`` exactly once. 

1918 

1919 For ``a = [a1, a2, a3]`` and ``b = [b1, b2, b3]``, an example of this 

1920 permutation type is ``a_perm = [a3, a1, a2]`` while ``b`` is left 

1921 in its original order. 

1922 

1923 ``permutation_type='pairings'`` supports ``data`` containing any number 

1924 of samples, each of which must contain the same number of observations. 

1925 All samples provided in ``data`` are permuted *independently*. Therefore, 

1926 if ``m`` is the number of samples and ``n`` is the number of observations 

1927 within each sample, then the number of permutations in an exact test is:: 

1928 

1929 factorial(n)**m 

1930 

1931 Note that if a two-sample statistic, for example, does not inherently 

1932 depend on the order in which observations are provided - only on the 

1933 *pairings* of observations - then only one of the two samples should be 

1934 provided in ``data``. This dramatically reduces computational cost without 

1935 affecting the shape of the null distribution (because the frequency/count 

1936 of each value is affected by the same factor). 

1937 

1938 **Paired statistics, permute samples** (``permutation_type='samples'``): 

1939 

1940 The null hypothesis associated with this permutation type is that 

1941 observations within each pair are drawn from the same underlying 

1942 distribution and that the sample to which they are assigned is random. 

1943 

1944 Suppose ``data`` contains two samples; e.g. ``a, b = data``. 

1945 Let ``n`` be the number of observations in ``a``, which must also equal 

1946 the number of observations in ``b``. 

1947 

1948 When ``1 < n_resamples < 2**n``, the elements of ``a`` are ``b`` are 

1949 randomly swapped between samples (maintaining their pairings) and the 

1950 statistic is calculated. This process is performed repeatedly, 

1951 `permutation` times, generating a distribution of the statistic under the 

1952 null hypothesis. The statistic of the original data is compared to this 

1953 distribution to determine the p-value. 

1954 

1955 When ``n_resamples >= 2**n``, an exact test is performed: the observations 

1956 are assigned to the two samples in each distinct way (while maintaining 

1957 pairings) exactly once. 

1958 

1959 For ``a = [a1, a2, a3]`` and ``b = [b1, b2, b3]``, an example of this 

1960 permutation type is ``x = [b1, a2, b3]`` and ``y = [a1, b2, a3]``. 

1961 

1962 ``permutation_type='samples'`` supports ``data`` containing any number 

1963 of samples, each of which must contain the same number of observations. 

1964 If ``data`` contains more than one sample, paired observations within 

1965 ``data`` are exchanged between samples *independently*. Therefore, if ``m`` 

1966 is the number of samples and ``n`` is the number of observations within 

1967 each sample, then the number of permutations in an exact test is:: 

1968 

1969 factorial(m)**n 

1970 

1971 Several paired-sample statistical tests, such as the Wilcoxon signed rank 

1972 test and paired-sample t-test, can be performed considering only the 

1973 *difference* between two paired elements. Accordingly, if ``data`` contains 

1974 only one sample, then the null distribution is formed by independently 

1975 changing the *sign* of each observation.

2260 """Perform Tukey's HSD test for equality of means over multiple treatments. 

2261 

2262 Tukey's honestly significant difference (HSD) test performs pairwise 

2263 comparison of means for a set of samples. Whereas ANOVA (e.g. `f_oneway`) 

2264 assesses whether the true means underlying each sample are identical, 

2265 Tukey's HSD is a post hoc test used to compare the mean of each sample 

2266 to the mean of each other sample. 

2267 

2268 The null hypothesis is that the distributions underlying the samples all 

2269 have the same mean. The test statistic, which is computed for every 

2270 possible pairing of samples, is simply the difference between the sample 

2271 means. For each pair, the p-value is the probability under the null 

2272 hypothesis (and other assumptions; see notes) of observing such an extreme 

2273 value of the statistic, considering that many pairwise comparisons are 

2274 being performed. Confidence intervals for the difference between each pair 

2275 of means are also available. 

2276 

2277 Parameters 

2278 ---------- 

2279 sample1, sample2, ... : array_like 

2280 The sample measurements for each group. There must be at least 

2281 two arguments. 

2282 

2283 Returns 

2284 ------- 

2285 result : `~scipy.stats._result_classes.TukeyHSDResult` instance 

2286 The return value is an object with the following attributes: 

2287 

2288 statistic : float ndarray 

2289 The computed statistic of the test for each comparison. The element 

2290 at index ``(i, j)`` is the statistic for the comparison between 

2291 groups ``i`` and ``j``. 

2292 pvalue : float ndarray 

2293 The computed p-value of the test for each comparison. The element 

2294 at index ``(i, j)`` is the p-value for the comparison between 

2295 groups ``i`` and ``j``. 

2296 

2297 The object has the following methods: 

2298 

2299 confidence_interval(confidence_level=0.95): 

2300 Compute the confidence interval for the specified confidence level.

2302 Notes 

2303 ----- 

2304 The use of this test relies on several assumptions. 

2305 

2306 1. The observations are independent within and among groups. 

2307 2. The observations within each group are normally distributed. 

2308 3. The distributions from which the samples are drawn have the same finite 

2309 variance. 

2310 

2311 The original formulation of the test was for samples of equal size [6]_. 

2312 In case of unequal sample sizes, the test uses the Tukey-Kramer method 

2313 [4]_.
