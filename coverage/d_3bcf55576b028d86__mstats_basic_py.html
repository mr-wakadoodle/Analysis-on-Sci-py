358def pearsonr(x, y): 

359 r""" 

360 Pearson correlation coefficient and p-value for testing non-correlation. 

361 

362 The Pearson correlation coefficient [1]_ measures the linear relationship 

363 between two datasets. The calculation of the p-value relies on the 

364 assumption that each dataset is normally distributed. (See Kowalski [3]_ 

365 for a discussion of the effects of non-normality of the input on the 

366 distribution of the correlation coefficient.) Like other correlation 

367 coefficients, this one varies between -1 and +1 with 0 implying no 

368 correlation. Correlations of -1 or +1 imply an exact linear relationship. 

369 

370 Parameters 

371 ---------- 

372 x : (N,) array_like 

373 Input array. 

374 y : (N,) array_like 

375 Input array. 

376 

377 Returns 

378 ------- 

379 r : float 

380 Pearson's correlation coefficient. 

381 p-value : float 

382 Two-tailed p-value. 

383 

384 Warns 

385 ----- 

386 PearsonRConstantInputWarning 

387 Raised if an input is a constant array. The correlation coefficient 

388 is not defined in this case, so ``np.nan`` is returned. 

389 

390 PearsonRNearConstantInputWarning 

391 Raised if an input is "nearly" constant. The array ``x`` is considered 

392 nearly constant if ``norm(x - mean(x)) < 1e-13 * abs(mean(x))``. 

393 Numerical errors in the calculation ``x - mean(x)`` in this case might 

394 result in an inaccurate calculation of r. 

395 

396 See Also 

397 -------- 

398 spearmanr : Spearman rank-order correlation coefficient. 

399 kendalltau : Kendall's tau, a correlation measure for ordinal data. 

400 

401 Notes 

402 ----- 

403 The correlation coefficient is calculated as follows: 

404 

405 .. math:: 

406 

407 r = \frac{\sum (x - m_x) (y - m_y)} 

408 {\sqrt{\sum (x - m_x)^2 \sum (y - m_y)^2}} 

409 

410 where :math:`m_x` is the mean of the vector x and :math:`m_y` is 

411 the mean of the vector y. 

412 

413 Under the assumption that x and y are drawn from 

414 independent normal distributions (so the population correlation coefficient 

415 is 0), the probability density function of the sample correlation 

416 coefficient r is ([1]_, [2]_): 

417 

418 .. math:: 

419 

420 f(r) = \frac{{(1-r^2)}^{n/2-2}}{\mathrm{B}(\frac{1}{2},\frac{n}{2}-1)} 

421 

422 where n is the number of samples, and B is the beta function. This 

423 is sometimes referred to as the exact distribution of r. This is 

424 the distribution that is used in `pearsonr` to compute the p-value. 

425 The distribution is a beta distribution on the interval [-1, 1], 

426 with equal shape parameters a = b = n/2 - 1. In terms of SciPy's 

427 implementation of the beta distribution, the distribution of r is:: 

428 

429 dist = scipy.stats.beta(n/2 - 1, n/2 - 1, loc=-1, scale=2) 

430 

431 The p-value returned by `pearsonr` is a two-sided p-value. The p-value 

432 roughly indicates the probability of an uncorrelated system 

433 producing datasets that have a Pearson correlation at least as extreme 

434 as the one computed from these datasets. More precisely, for a 

435 given sample with correlation coefficient r, the p-value is 

436 the probability that abs(r') of a random sample x' and y' drawn from 

437 the population with zero correlation would be greater than or equal 

438 to abs(r). In terms of the object ``dist`` shown above, the p-value 

439 for a given r and length n can be computed as:: 

440 

441 p = 2*dist.cdf(-abs(r)) 

442 

443 When n is 2, the above continuous distribution is not well-defined. 

444 One can interpret the limit of the beta distribution as the shape 

445 parameters a and b approach a = b = 0 as a discrete distribution with 

446 equal probability masses at r = 1 and r = -1. More directly, one 

447 can observe that, given the data x = [x1, x2] and y = [y1, y2], and 

448 assuming x1 != x2 and y1 != y2, the only possible values for r are 1 

449 and -1. Because abs(r') for any sample x' and y' with length 2 will 

450 be 1, the two-sided p-value for a sample of length 2 is always 1. 

451 

452 References 

453 ---------- 

454 .. [1] "Pearson correlation coefficient", Wikipedia, 

455 https://en.wikipedia.org/wiki/Pearson_correlation_coefficient 

456 .. [2] Student, "Probable error of a correlation coefficient", 

457 Biometrika, Volume 6, Issue 2-3, 1 September 1908, pp. 302-310. 

458 .. [3] C. J. Kowalski, "On the Effects of Non-Normality on the Distribution 

459 of the Sample Product-Moment Correlation Coefficient" 

460 Journal of the Royal Statistical Society. Series C (Applied 

461 Statistics), Vol. 21, No. 1 (1972), pp. 1-12. 

462 

463 Examples 

464 -------- 

465 >>> from scipy import stats 

466 >>> from scipy.stats import mstats 

467 >>> mstats.pearsonr([1, 2, 3, 4, 5], [10, 9, 2.5, 6, 4]) 

468 (-0.7426106572325057, 0.1505558088534455) 

469 

470 There is a linear dependence between x and y if y = a + b*x + e, where 

471 a,b are constants and e is a random error term, assumed to be independent 

472 of x. For simplicity, assume that x is standard normal, a=0, b=1 and let 

473 e follow a normal distribution with mean zero and standard deviation s>0. 

474 

475 >>> s = 0.5 

476 >>> x = stats.norm.rvs(size=500) 

477 >>> e = stats.norm.rvs(scale=s, size=500) 

478 >>> y = x + e 

479 >>> mstats.pearsonr(x, y) 

480 (0.9029601878969703, 8.428978827629898e-185) # may vary 

481 

482 This should be close to the exact value given by 

483 

484 >>> 1/np.sqrt(1 + s**2) 

485 0.8944271909999159 

486 

487 For s=0.5, we observe a high level of correlation. In general, a large 

488 variance of the noise reduces the correlation, while the correlation 

489 approaches one as the variance of the error goes to zero. 

490 

491 It is important to keep in mind that no correlation does not imply 

492 independence unless (x, y) is jointly normal. Correlation can even be zero 

493 when there is a very simple dependence structure: if X follows a 

494 standard normal distribution, let y = abs(x). Note that the correlation 

495 between x and y is zero. Indeed, since the expectation of x is zero, 

496 cov(x, y) = E[x*y]. By definition, this equals E[x*abs(x)] which is zero 

497 by symmetry. The following lines of code illustrate this observation: 

498 

499 >>> y = np.abs(x) 

500 >>> mstats.pearsonr(x, y) 

501 (-0.016172891856853524, 0.7182823678751942) # may vary 

502 

503 A non-zero correlation coefficient can be misleading. For example, if X has 

504 a standard normal distribution, define y = x if x < 0 and y = 0 otherwise. 

505 A simple calculation shows that corr(x, y) = sqrt(2/Pi) = 0.797..., 

506 implying a high level of correlation: 

507 

508 >>> y = np.where(x < 0, x, 0) 

509 >>> mstats.pearsonr(x, y) 

510 (0.8537091583771509, 3.183461621422181e-143) # may vary 

511 

512 This is unintuitive since there is no dependence of x and y if x is larger 

513 than zero which happens in about half of the cases if we sample x and y. 

514 """

668def _kendall_p_exact(n, c, alternative='two-sided'): 

669 

670 # Use the fact that distribution is symmetric: always calculate a CDF in 

671 # the left tail. 

672 # This will be the one-sided p-value if `c` is on the side of 

673 # the null distribution predicted by the alternative hypothesis. 

674 # The two-sided p-value will be twice this value. 

675 # If `c` is on the other side of the null distribution, we'll need to 

676 # take the complement and add back the probability mass at `c`. 

677 in_right_tail = (c >= (n*(n-1))//2 - c) 

678 alternative_greater = (alternative == 'greater') 

679 c = int(min(c, (n*(n-1))//2 - c))

682 # (4th Edition), Charles Griffin & Co., 1970. 

683 if n <= 0: 683 ↛ 684line 683 didn't jump to line 684, because the condition on line 683 was never true

688 prob = 1.0 

689 p_mass_at_c = 1 

690 elif n == 2:

692 p_mass_at_c = 0.5 

693 elif c == 0: 

694 prob = 2.0/math.factorial(n) if n < 171 else 0.0 

695 p_mass_at_c = prob/2 

696 elif c == 1: 

697 prob = 2.0/math.factorial(n-1) if n < 172 else 0.0 

698 p_mass_at_c = (n-1)/math.factorial(n) 

699 elif 4*c == n*(n-1) and alternative == 'two-sided': 

700 # I'm sure there's a simple formula for p_mass_at_c in this 

701 # case, but I don't know it. Use generic formula for one-sided p-value. 

702 prob = 1.0 

703 elif n < 171: 

704 new = np.zeros(c+1) 

705 new[0:2] = 1.0 

706 for j in range(3,n+1):

708 if j <= c: 

709 new[j:] -= new[:c+1-j] 

710 prob = 2.0*np.sum(new)/math.factorial(n) 

711 p_mass_at_c = new[-1]/math.factorial(n) 

712 else:

723 # if the alternative hypothesis and alternative agree, 

724 # one-sided p-value is half the two-sided p-value

728 prob = 1 - prob/2 + p_mass_at_c
